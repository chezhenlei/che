1. 说说 JavaScript 的事件循环（Event Loop）JS

JavaScript 是单线程语言，为了处理异步任务，浏览器引入了事件循环机制。整个执行过程如下：

  • 执行栈：同步代码在执行栈中按顺序执行
  • 微任务队列（Microtask Queue）：Promise.then、MutationObserver、queueMicrotask、Async/Await（await 之后的代码）
  • 宏任务队列（Macrotask Queue）：setTimeout、setInterval、I/O、UI 渲染、requestAnimationFrame

执行顺序：执行完当前执行栈 → 清空所有微任务 → 执行一个宏任务 → 重复循环。浏览器会在宏任务之间插入渲染（每 16.6ms 一次）。

示例： console.log('1'); // 同步 setTimeout(() => console.log('2'), 0); // 宏任务 Promise.resolve().then(() => console.log('3')); // 微任务 console.log('4'); // 同步 // 输出顺序：1 → 4 → 3 → 2
2. 闭包是什么？有什么应用和风险？JS

闭包：函数能够记住并访问其词法作用域，即使该函数在其词法作用域之外被调用。

常见应用场景：

  • 模块化私有变量：创建私有作用域，封装不想暴露的变量
  • 防抖（debounce）：延迟执行，n 毫秒内重复触发重新计时
  • 节流（throttle）：限制执行频率，n 毫秒内只执行一次
  • 函数柯里化：预置参数，形成参数复用
  • 缓存/记忆化：存储计算结果避免重复计算

潜在风险：

  • 未及时清理的定时器、事件监听器会导致内存泄漏
  • 在循环中创建闭包可能产生意外行为（需用 let 声明块级作用域）
3. 谈谈 this 的指向规则JS

this 的指向遵循以下优先级（从高到低）：

  • 1. new 绑定：new 调用构造函数，this 指向新创建的对象
  • 2. 显式绑定：call、apply、bind 显式指定 this
  • 3. 隐式绑定：作为对象的方法调用，this 指向该对象
  • 4. 默认绑定：严格模式下 this 为 undefined，非严格模式下为 window/global

箭头函数：没有自己的 this，沿用外层函数的 this（词法绑定），call/apply/bind 无法改变。

const obj = { name: 'test', fn: () => this }; // this 指向外层 const obj2 = { name: 'test', fn() { return this; } }; // this 指向 obj2
4. Promise、async/await 与错误处理JS

Promise：三种状态 pending → fulfilled/rejected（不可逆）。then 返回新 Promise，catch 返回新的 Promise（除非抛出错误）。

async/await：async 函数自动返回 Promise，await 等待 Promise 结果，使异步代码看起来像同步。

错误处理方式：

  • try/catch：可捕获同步和 async 函数中的错误
  • Promise.catch：捕获 Promise 链中的错误
  • window.onerror / unhandledrejection：全局捕获未处理的错误

并发请求：

  • Promise.all：全部成功返回结果数组，一个失败则整体 reject
  • Promise.allSettled：全部执行完，返回每个 Promise 的状态和结果
  • Promise.race：返回最先完成（成功或失败）的 Promise
  • Promise.any：返回最先成功的结果，全部失败才 reject
5. 防抖与节流的区别及实现思路JS

防抖（debounce）：n 毫秒内重复触发，重新计算时间，只执行最后一次。常用于搜索框输入、窗口调整大小。

节流（throttle）：n 毫秒内只执行一次，冷却期内忽略后续调用。常用于滚动事件、按钮防双击。

防抖实现： function debounce(fn, delay) { let timer = null; return function(...args) { clearTimeout(timer); timer = setTimeout(() => { fn.apply(this, args); }, delay); }; } 节流实现： function throttle(fn, delay) { let lastTime = 0; return function(...args) { const now = Date.now(); if (now - lastTime >= delay) { lastTime = now; fn.apply(this, args); } }; }

7. 原型链与继承的实现方式JS

每个对象都有 __proto__ 属性指向其构造函数的 prototype，形成原型链。属性查找沿原型链向上直到 Object.prototype 为止。

继承实现方式：

  • 原型链继承：子类的 prototype 指向父类实例
  • 构造函数继承：在子类中调用父类.call(this)
  • 组合继承：原型链 + 构造函数，双重调用
  • 原型式继承：Object.create() 创建对象
  • ES6 Class：extends 语法糖，底层仍是原型链
class Child extends Parent { constructor(name, age) { super(); // 调用父类构造函数 this.name = name; this.age = age; } }
8. CSS 盒模型与 BFCCSS

盒模型：

  • 标准盒模型：width = content，padding + border + margin 另算
  • 怪异盒模型（IE）：width = content + padding + border

通过 box-sizing: border-box 切换为 IE 模型。

BFC（块级格式化上下文）：独立的渲染区域，不会影响外部元素。触发条件：

  • overflow 不为 visible
  • display: flow-root / inline-block / table
  • position: absolute / fixed
  • float 不为 none

BFC 特性：清除浮动、阻止外边距合并、阻止元素被浮动元素覆盖。

12. 浏览器渲染流程与重排重绘性能

渲染流程：

  1. HTML 解析 → DOM 树
  2. CSS 解析 → CSSOM 树
  3. DOM + CSSOM → Render Tree
  4. Layout（计算几何信息）
  5. Paint（绘制图层）
  6. Composite（合成图层，上传 GPU）

重排（Reflow）：几何属性变化（宽高、位置、显示隐藏），成本高。

重绘（Repaint）：外观变化不影响布局（颜色、背景），成本较低。

优化策略：

  • 批量修改 DOM（先 display:none，改完再显示）
  • 使用 transform/opacity 触发 GPU 加速，走合成层，不触发重排重绘
  • 避免频繁读取布局信息（offsetHeight 等），必要时缓存
  • 使用 requestAnimationFrame 替代 setTimeout 修改动画

14. 从输入 URL 到页面展示发生了什么？网络
  1. DNS 解析：域名 → IP 地址（浏览器缓存 → 系统缓存 → hosts → DNS 服务器）
  2. TCP 三次握手：建立可靠连接（SYN → SYN-ACK → ACK）
  3. TLS/SSL 握手（HTTPS）：协商加密算法，验证服务器证书
  4. 发送 HTTP 请求：请求行、请求头、请求体
  5. 服务器处理：处理请求，返回 HTTP 响应
  6. 浏览器解析：解析 HTML 构建 DOM 树，解析 CSS 构建 CSSOM
  7. 渲染树 + 布局：计算几何信息
  8. 绘制：分层绘制页面
  9. 执行 JS：JS 可能触发 DOM 操作，重新触发渲染流程
  10. TCP 四次挥手：连接关闭

每一步都可能成为性能瓶颈，需结合具体场景分析优化。

15. HTTP 缓存机制网络

强缓存（不发请求，直接使用本地缓存）：

  • Cache-Control: max-age=31536000（相对时间，优先级高于 Expires）
  • Expires: Wed, 21 Oct 2026 07:28:00 GMT（绝对时间，已淘汰）
  • 命中返回 200 (from cache/disk/memory)

协商缓存（发请求验证资源是否更新）：

  • Last-Modified / If-Modified-Since（时间戳，精度到秒）
  • ETag / If-None-Match（文件指纹，精度更高，可解决时间戳问题）
  • 命中返回 304 Not Modified

推荐策略：

  • 静态资源（js/css/图片）：强缓存 + hash 文件名
  • HTML 文件：协商缓存，确保更新能及时生效
  • API 数据：no-cache 或较短的 max-age
16. 跨域问题与解决方案网络

同源策略：协议、域名、端口任一不同即为跨域。限制 AJAX 请求、Cookie、LocalStorage、iframe 访问。

解决方案：

  • CORS（推荐）：后端设置响应头 Access-Control-Allow-Origin: * 简单请求预检直接通过，复杂请求先发 OPTIONS 预检
  • Nginx/Node 反向代理：同域代理，配置 location 转发请求到目标服务器
  • JSONP（仅 GET，已少用）：利用 script 标签不受同源限制，需后端配合返回回调
  • postMessage：跨窗口通信，如 iframe 与父页面
  • WebSocket：不受同源策略限制
  • document.domain：同主域的子域之间可设置
17. 常见的 Web 安全问题安全

XSS（跨站脚本攻击）：

  • 攻击者注入恶意脚本到页面
  • 防护：输入输出转义、CSP（Content-Security-Policy）限制脚本来源、HttpOnly Cookie、开启 X-XSS-Protection

CSRF（跨站请求伪造）：

  • 用户已登录，诱导请求目标站点，自动携带 Cookie
  • 防护：Token 校验、SameSite Cookie、验证 Referer/Origin、敏感操作二次验证

点击劫持（Clickjacking）：

  • 透明 iframe 覆盖在页面上，诱导用户点击
  • 防护：X-Frame-Options: DENY/SAMEORIGIN、CSP frame-ancestors

其他：SQL 注入（参数化查询）、中间人攻击（HTTPS）、敏感数据泄露（加密存储）

18. HTTPS 与 HTTP 的区别网络

HTTPS = HTTP + TLS/SSL，核心区别在于安全性：

  • 加密传输：数据加密，防窃听、篡改
  • 身份认证：证书验证服务器身份，防止中间人攻击
  • 数据完整性：MAC 校验，防数据被篡改

TLS 握手流程（简述）：

  1. 客户端发送支持的加密套件列表、随机数
  2. 服务器返回证书（包含公钥）、选定的加密套件、随机数
  3. 客户端验证证书有效性，用公钥加密预主密钥发送
  4. 双方用预主密钥生成会话密钥，后续数据加密传输

HTTPS 缺点：增加 TLS 握手延迟（1-2 RTT）、证书费用、需要服务器资源。可通过 HTTP/2、CDN、证书缓存优化。

20. React 函数组件与 Hooks 注意事项React

Hooks 规则：

  • 只能在组件顶层调用，不能放在条件、循环、嵌套函数中
  • 只能在 React 函数组件或自定义 Hook 中调用

useEffect 注意事项：

  • 处理副作用，在返回函数中清理（定时器、事件监听）
  • 依赖数组要完整，避免遗漏依赖导致闭包陷阱
  • 可使用 ESLint exhaustive-deps 规则检查

状态相关：

  • setState 是异步合并的，如需依赖上一次状态，用函数式 setState
  • 对象状态要展开合并：setState(prev => ({ ...prev, ...updates }))
  • useReducer 适合复杂状态逻辑

性能优化：

  • useMemo：缓存计算结果
  • useCallback：缓存函数引用，配合 React.memo 避免子组件不必要的重渲染
  • 不要过度优化，优先保证代码可读性

22. 受控组件与非受控组件React

受控组件：表单值由 React 状态驱动，通过 onChange 更新状态。

const [value, setValue] = useState(''); setValue(e.target.value)} />

优点：值可控、便于校验、与其他状态联动、即时响应

缺点：每次输入触发状态更新，可能有性能开销

非受控组件：通过 ref 直接操作 DOM 读取值。

const inputRef = useRef(); const handleSubmit = () => console.log(inputRef.current.value);

适用场景：简单表单、与第三方库集成、文件上传控件

建议：表单较多时使用 react-hook-form 或 Formik，减少受控组件的样板代码。

23. React 与 Vue 的核心区别ReactVue

响应式原理：

  • Vue：基于数据劫持 + 订阅发布，自动追踪依赖变化，修改数据自动更新视图
  • React：基于虚拟 DOM + diff，手动调用 setState 触发更新，重新渲染组件树

模板 vs JSX：

  • Vue：模板语法，提供指令（v-if、v-for），对新手友好
  • React：JSX，JavaScript 表达式，灵活但模板和逻辑混杂

CSS 方案：

  • Vue：scoped CSS、CSS Modules 原生支持
  • React：CSS-in-JS（styled-components、emotion）或 CSS Modules

生态与社区：React 生态更大、更灵活；Vue 上手更快、文档更友好。

  • Composition API：逻辑复用更灵活，同一逻辑集中管理，TypeScript 支持更好
  • 性能提升：虚拟 DOM 重写，编译时优化（静态提升、缓存事件处理器）
  • Tree Shaking：按需引入，减小打包体积
  • 更好的 TypeScript 支持：Vue3 本身用 TS 重写
  • 新的响应式系统：Proxy 替代 Object.defineProperty，解决无法监听新增属性、数组下标问题
  • Suspense：异步组件加载状态
  • Teleport：将组件 DOM 传送到指定位置
  • 多个根节点：模板可有多根节点，无需包裹 div
  • 自定义渲染器：可自定义 render 函数

27. CI/CD 与自动化部署工程化

CI（持续集成）：频繁合并代码到主线，自动构建和测试，尽早发现 bug。

CD（持续交付/部署）：自动化发布流程，快速、安全地将代码部署到各环境。

前端 CI/CD 流程：

  1. 代码提交 → 触发 Webhook
  2. 拉取代码 → 安装依赖（npm ci 锁定版本）
  3. 运行检查：ESLint、Lint-staged、单元测试
  4. 构建：Webpack/Vite 打包
  5. 部署：上传 CDN / 构建镜像 / 滚动更新
  6. 通知：钉钉/Slack 通知结果

常用工具：GitHub Actions、GitLab CI、Jenkins、Travis CI、CircleCI

28. 前端监控与异常追踪性能

为什么要监控：及时发现线上问题，量化优化效果，了解用户真实体验。

监控指标：

  • Performance API：FP、FCP、LCP、CLS、INP、FID
  • 前端异常：JS Error、Promise Rejection、Resource Load Error
  • 业务数据：接口成功率、接口响应时间、白屏时间
  • 用户行为：PV/UV、点击热力图、页面停留时长

实现方案：

  • PerformanceObserver 监听性能指标
  • window.onerror + unhandledrejection 捕获异常
  • 上报服务（可自建或用 Sentry、FunDebug）
  • 采集用户关键操作路径（埋点）

Sentry：开源的异常追踪平台，支持 SourceMap、关联 Release、告警通知。

29. 说说对前端工程化的理解工程化

前端工程化是将软件工程的方法和思想应用到前端开发，提高效率、保证质量、降低维护成本。

核心领域：

  • 模块化：ES Modules、CommonJS、AMD
  • 组件化：UI 组件、业务组件、可复用单元
  • 规范化：ESLint、Prettier、Git Hooks（Lint-staged）、Commit 规范
  • 自动化：CI/CD、自动化测试、自动化部署
  • 工具化：构建工具（Webpack/Vite/Rollup）、脚手架

质量保障：

  • 单元测试（Jest/Vitest）、E2E 测试（Playwright/Cypress）
  • Code Review
  • 灰度发布 + 回滚机制
30. 项目难点与团队协作经验（开放题）软实力

建议采用 STAR 法则（Situation-Task-Action-Result）回答：

示例 1：旧项目升级 背景：旧项目 Webpack 4 构建时间 60s+，严重影响开发效率 方案：升级到 Vite，基于 ESM 利用浏览器缓存，二次构建 5s 内 结果：开发体验大幅提升，团队满意度显著提高
示例 2：性能优化 背景：首屏加载 5s+，用户流失率高 方案：分包 + 懒加载 + 图片压缩 + SSR 预渲染 结果：LCP 从 5s 降至 1.5s，转化率提升 20%
示例 3：组件库建设 背景：多项目重复开发，维护成本高 方案：基于 Ant Design 二次封装，沉淀 50+ 业务组件 结果：新人上手时间从 2 周缩短到 3 天，开发效率提升 40%

加分项：

  • 带新人 / Code Review 规范的落地
  • 线上事故复盘与改进
  • 推动技术债务治理
  • 团队技术分享与知识沉淀

重点：突出业务理解、推动力、技术视野与可衡量的产出。
