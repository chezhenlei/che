前端面试题汇总（20题）

适用：5 年以上前端 / 中小型公司 / 外包项目 · 涵盖 JS、CSS、React、性能、网络、工程化

1. 说说 JavaScript 的事件循环（Event Loop）JS
JS 是单线程，主线程执行同步代码后会先清空所有微任务队列（Promise.then、MutationObserver、queueMicrotask），再执行一个宏任务（setTimeout、setInterval、I/O、UI 渲染等），如此循环。浏览器还会在合适时机插入渲染。
2. 闭包是什么？有什么应用和风险？JS
闭包是函数与其词法作用域的引用绑定。常见用途：模块私有变量、防抖节流、柯里化、函数式封装。风险：未释放的引用会造成内存泄漏，需注意定时器、事件监听及时解绑。
3. 谈谈 this 的指向规则JS
优先级：new > bind/call/apply > 对象方法调用 > 默认调用（严格模式 undefined，非严格 window）。箭头函数无自己的 this，沿用外层词法 this。
4. Promise、async/await 与错误处理JS
Promise 三态不可逆，then 返回新 Promise；async 函数返回 Promise，await 等待结果。错误捕获用 try/catch，或在 Promise 链末尾加 catch；并发用 Promise.all，全部结果用 Promise.allSettled。
5. 防抖与节流的区别及实现思路JS
防抖（debounce）：n 毫秒内重复触发，重新计时，最后一次执行，常用于搜索输入。节流（throttle）：n 毫秒内只执行一次，常用于滚动、resize。实现核心是闭包保存 timer/lastTime。
6. ES6+ 你常用的哪些特性？JS
let/const、解构、模板字符串、箭头函数、Class、Promise/async、Map/Set、可选链 ?.、空值合并 ??、ES Modules、Symbol、迭代器/生成器等。
7. CSS 盒模型与 BFCCSS
标准盒模型 width 仅含内容区；IE/怪异盒模型 width 含 padding+border，可用 box-sizing: border-box 切换。BFC 是块级格式化上下文，可清除浮动、阻止外边距合并，触发条件如 overflow 非 visible、display:flow-root。
8. Flex 与 Grid 的使用场景CSS
Flex 适合一维布局（导航栏、按钮组、垂直居中），Grid 适合二维布局（页面整体结构、复杂卡片）。两者可组合使用：Grid 做骨架，Flex 做局部对齐。
9. 实现垂直水平居中的几种方式CSS
  • flex：display:flex; justify-content:center; align-items:center;
  • grid：display:grid; place-items:center;
  • 绝对定位 + transform：top:50%;left:50%;transform:translate(-50%,-50%);
  • inline-block + text-align + line-height（旧方案）
10. 浏览器渲染流程与重排重绘性能
流程：解析 HTML→DOM、解析 CSS→CSSOM、合成 Render Tree→Layout→Paint→Composite。修改尺寸/位置触发重排（reflow），仅改颜色等触发重绘。优化：批量修改、用 transform/opacity 走合成层、避免强制同步布局。
11. 从输入 URL 到页面展示发生了什么？网络
DNS 解析 → TCP 三次握手（HTTPS 还需 TLS 握手）→ 发送 HTTP 请求 → 服务器响应 → 浏览器解析渲染 → 执行 JS → 触发后续接口 → 释放连接。每一步都有可能成为性能瓶颈。
12. HTTP 缓存机制网络
分强缓存（Cache-Control: max-age、Expires，命中返回 200 from cache）和协商缓存（Last-Modified/If-Modified-Since、ETag/If-None-Match，命中返回 304）。静态资源用强缓存+hash 文件名，HTML 用协商缓存。
13. 跨域问题与解决方案网络
同源策略限制不同协议/域名/端口的请求。解决：CORS（推荐，由后端配置响应头）、Nginx/Node 反向代理、JSONP（仅 GET，已少用）、postMessage（跨窗口通信）、document.domain（同主域）。
14. 常见的 Web 安全问题安全
XSS：输入输出做转义、CSP 限制脚本来源；CSRF：Token 校验、SameSite Cookie、验证 Referer；点击劫持：X-Frame-Options；接口安全：HTTPS、签名、限频。
15. React 函数组件与 Hooks 注意事项React
Hooks 必须在组件顶层调用，不能放在条件/循环里；useEffect 中处理副作用并在返回函数里清理；依赖数组要写完整；useMemo/useCallback 用于稳定引用，不要滥用；状态更新是异步合并的。
16. React 性能优化手段React
合理拆分组件、使用 memo/useMemo/useCallback、列表加 key、避免在 render 里创建新引用、按需引入、路由懒加载（React.lazy + Suspense）、虚拟列表处理长列表、避免不必要的 Context 引发全树渲染。
17. 受控组件与非受控组件React
受控组件：表单值由 React 状态驱动，便于校验和联动；非受控组件：通过 ref 直接读取 DOM 值，适合简单场景或与第三方库集成。表单较多时可借助 react-hook-form、Formik 等。
18. 前端首屏性能优化思路性能
  • 资源：压缩、Gzip/Brotli、分包、Tree Shaking、CDN、HTTP/2
  • 加载：路由懒加载、图片懒加载、关键资源 preload
  • 渲染：SSR/预渲染、骨架屏、避免大体积同步 JS
  • 缓存：强缓存 hash 文件、Service Worker
  • 监控：LCP/FCP/CLS/INP 指标驱动迭代
19. Webpack/Vite 构建优化经验工程化
Webpack：splitChunks 拆包、缓存（cache、HardSource）、thread-loader 多进程、DllPlugin/外部化大库、source-map 选择合适模式。Vite：基于 ESM dev 启动快，生产 Rollup 打包；优化预构建依赖、合理使用插件、按需加载。
20. 项目难点与团队协作经验（开放题）软实力
建议结构：背景 → 难点 → 你的方案 → 量化结果。例如：旧项目升级（Webpack→Vite，构建从 60s→8s）；多端组件库沉淀（提效 30%）；线上事故复盘（监控 + 报警 + 兜底）；带新人 / Code Review 规范的落地等。重点突出业务理解、推动力与可衡量的产出。
